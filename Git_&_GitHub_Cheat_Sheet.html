Git & GitHub Cheat Sheet

This is a cheat sheet containing most important and commonly used Git commands.

What is Git?

  • Git is a free open-source distributed version control system that allows developers to keep track of their changes made to their coding files.
  • It also allows developers to collaborate with other developers and work along on a project.

What is Version Control System?

  • A version control system(VCS) is a system which records all the changes made to a file. So in future if any version of the file is required, it can be called.
  • VCS also ensures that all the team members are working on the latest version of the file.

What is GitHub?

  • GitHub is an open-source hosting platform where developers host their projects, store their codes, and so on.
  • It also allows other developers to collaborate on the projects.

Commands:-

  • git config --global user.name "firstName middleName lastName"
    to set a user name
  • git config --global user.email "valid emailID"
    to set a user valid emailID
  • git config --global user.name
    to check user name
  • git config --global user.email
    to check user valid emailID

  • pwd
    to know present working directory
  • cd "directoryName"
    to change directory
  • cd ..
    to go back to a previous directory

  • git
    provides a list of git commands
  • git config --list
    to show all of the different values assigned to variables.

  • vim
    to open vim editor
  • :q
    to close vim editor
  • git config --global core.editor emacs
    to open emacs editor on Linux OS. It will not open on Windows OS.
  • git config --global core.editor vim
    to open vim editor on Linux OS. It will not open on Windows OS.

  • git init
    initialize the current directory as a Git repository
  • git init <repo name>
    create empty Git repo in specified directory
  • mkdir <directoryName>
    to make a directory in the current directory
  • touch <fileName>
    to make a file in the current directory

  • git status
    to show modified files in the current working directory, staged for your next commit. List files which are staged, unstaged, and untracked.
  • git add <fileName or directoryName>
    to staged a particular file or directory in the staging area for the next commit
  • git add --a
    to staged all the modified files in the staging area for the next commit
  • git commit <fileName or directoryName> -m "your commit message"
    to commit a particular file or directory of the staging area. Commit message is a string so keep it in double quotation.
  • git commit -m "your commit message"
    to commit all the staged files or directories of the staging area. Commit message is a string so keep it in double quotation.
  • git log
    to show all the previous commits and their details
