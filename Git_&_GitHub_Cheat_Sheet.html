Git & GitHub Cheat Sheet

This is a cheat sheet containing most important and commonly used Git commands.

What is Git?

  • Git is a free open-source distributed version control system that allows developers to keep track of their changes made to their coding files.
  • It also allows developers to collaborate with other developers and work along on a project.

What is Version Control System?

  • A version control system(VCS) is a system which records all the changes made to a file. So in future if any version of the file is required, it can be called.
  • VCS also ensures that all the team members are working on the latest version of the file.

What is GitHub?

  • GitHub is an open-source hosting platform where developers host their projects, store their codes, and so on.
  • It also allows other developers to collaborate on the projects.

Commands:-

  • git config --global user.name "firstName middleName lastName"
    to set a user name
  • git config --global user.email "valid emailID"
    to set a user valid emailID
  • git config --global user.name
    to check user name
  • git config --global user.email
    to check user valid emailID

  • pwd
    to know present working directory
  • cd <directoryName>
    to change directory
  • cd ..
    to go back to a previous directory

  • git
    provides a list of git commands
  • git config --list
    to show all of the different values assigned to variables.

  • vim
    to open vim editor
  • :q
    to close vim editor
  • git config --global core.editor emacs
    to open emacs editor on Linux OS. It will not open on Windows OS.
  • git config --global core.editor vim
    to open vim editor on Linux OS. It will not open on Windows OS.

  • git init
    initialize the current directory as a Git repository
  • git init <repo name>
    create empty Git repo in the specified directory
  • mkdir <directoryName>
    to make a directory in the current directory
  • touch <fileName>
    to make a file in the current directory

  • git status
    to show modified files in the current working directory, staged for your next commit. List files which are staged, unstaged, and untracked.
  • git add <fileName or directoryName>
    to staged a particular file or directory in the staging area for the next commit
  • git add --a or git add .
    to staged all the modified files in the staging area for the next commit
  • git commit <fileName or directoryName> -m "your commit message"
    to commit a particular file or directory of the staging area. Commit message is a string so keep it in double quotation.
  • git commit -m "your commit message"
    to commit all the staged files or directories of the staging area. Commit message is a string so keep it in double quotation.
  • git commit -a -m "your commit message"
    to skip the staging area and commit directly. Commit message is a string so keep it in double quotation.
  • git log
    to show all the previous commits and their details

  • press up arrow key
    to copy previous or above commands
  • exit
    to exit from the git bash terminal

  • ls
    list all the files present in the current directory
  • rm <fileName>
    to delete a file from the current directory
  • rm -rf <directoryName>
    to delete a directory from the current directory
  • git rm --cached <fileName>
    to make a file untracked from tracked. Removes the specified file from the staging area, but the file remains in your local working directory as an untracked file. Git will no longer track changes to this file.

  • git clone <URL> <directoryName>
    to clone a remote repository in the specified directory

  • .gitignore
    contains all the ignored files or directories
  • /directoryName/
    to ignore a particular directory. Write the /directory name/ in .gitignore.

  • git diff
    shows unstaged changes between the working directory and index. It shows what is changed but not staged.
  • git diff --staged
    shows what is staged but not yet committed

  • git mv <existing fileName> <new fileName>
    to change an existing fileName to a new newName

  • rm -rf .git
    It is a dangerous command as it can delete .git directory resulting in losing all the history of the project.

  • git log -p
    to show all the previous commits, their details along with the changes introduced in each commit.
  • git log -n
    n is any number. It shows a list of n most recent commits made to a repository. For exmaple, git log -2 will show the last two commits.
  • git log --stat
    to show the files that were modified in each commit, the number of lines added, removed, and a summary line with the total number of files and lines changed.
  • git log --pretty=oneline
    to show each commit in a single line resulting in taking less space. Each line representing a commit and its associated commit hash and commit message.
  • git log --pretty=short
    to show each commit in short. It shows each commit containing commit hash, author name, and commit message.
  • git log --pretty=full
    it shows each commit containing commit hash, author name, commit name(the person who committed) and commit message.
  • git log --since=<date>
    to show the commit history by the date range. For example, git log --since=2.days/months/years will show the commits of last two days/two months/two years respectively.
  • git log --pretty=format:"custom format"
    to show commit history in the specified format. Format should be inside double quotation. For example, git log --pretty=format:"%h"/"%h--%an"/"%h--%ae" will show only commit hash/commit hash and author name/commit hash and author emailID respectively.

  • git commit --amend
    to modify the most recent commit. It combines the changes in the staging area with the last commit and creates a new commit.

  • git restore --staged <fileName>
    to unstage the staged file.
  • git checkout -- <fileName>
    to restore the previous version of the file. It will replace the version of the file we have in the working directory with the one we have in the staging area.
  • git checkout -f
    to restore the previous versions of all the modified files.

  • git remote add <name> <URL>
    to add a new remote repository to a Git repository. After adding a remote, <name> can be used as a shortcut for <URL> in other commands.
  • git remote -v
    it shows the URLs that git has stored along with their shortcut names which is used to read and write to that remote.
  • git push -u origin main
    to push the local branch named "main" to the remote repository named "origin"
  • git branch -M main
    to rename the current working branch to main or any preferred name
  • git push
    to update the remote branch with the local commits or changes

  • git config --global alias.<aliasName> <gitCommand>
    to create a shortcut name for Git Commands
    For example:
    • git config --global alias.st status - git status/git st
    • git config --global alias.cm commit - git commit/git cm
    • git config --global alias.unstage 'restore --staged --' - git restore --staged <fileName>/git unstage <fileName>
    • git config --global alias.last 'log -p -1' - git log -p -1/git last

  • git restore <fileName>
    to restore the previoss version of the specified file as they were in the last commit.
  • git branch
    list all the branches in the current repository. * will appear next to the currently active branch.
  • git checkout -b <branchName>
    to create a new branch named <branchName>
  • git checkout <branchName>
    to switch to the branch named <branchName>
